# 30日間で学ぶGo言語入門カリキュラム（解説駆動型）

### 本カリキュラムの目的と約束

このカリキュラムは、単なる学習項目のリストではありません。各ステップにおいて、**「なぜそうするのか？」という背景や概念を私が直接解説する**ことを前提として設計されています。

学習者が「調べる」手間を最小限にし、対話を通じて理解を深めることを目的とします。今後のやり取りは、この方針に沿って進められます。

---

### 概要

1日1〜2時間の学習を想定し、30日間でGo言語の基礎から簡単なWebアプリケーションを構築するまでを目標とします。

---

## Week 1: Go言語の基礎固め（必須基礎）

**目標:** Goの基本構文を理解し、簡単なコマンドラインプログラムを作成できる。

- **Day 1-2: 環境構築とHello, World**
  - **学習項目:**
    - **Goのインストールと環境設定:**
      - 公式サイトからGoをインストールし、ターミナルで `go version` を実行して正しくインストールされたか確認します。
      - **GOROOTとGOPATHの理解:**
        - **GOROOT:** Goがインストールされた場所を指す環境変数です。Goのコンパイラや標準ライブラリが含まれています。通常はGoのインストーラが自動で設定するため、手動での設定は不要です。
        - **GOPATH:** Goのワークスペースとして使われる場所です。現在は主に`go install`でインストールしたツールなどが保存される場所として機能します。手動で設定しなくても自動でデフォルトの場所が使われるため、意識する必要はほとんどありません。
    - **"Hello, World!" プログラムの作成と実行:**
      - 好きな場所にプロジェクト用のディレクトリを作成します。
      - `hello.go`というファイルを作成し、`go run hello.go`で実行します。
      - **`go run`と`go build`の違い:**
        - `go run`: 開発中に素早く動作確認するためのコマンドです。コンパイルと実行を一度に行いますが、実行ファイルは保存されません。
        - `go build`: プログラムを配布・実行したいときに使うコマンドです。コンパイルして実行可能なバイナリファイル（Windowsなら`.exe`ファイル）を生成します。このファイルはGoがインストールされていない環境でも動作します。
    - **Goの基本的なプログラム構造:**
      - Goの実行可能なプログラムは`package main`という宣言から始まります。
      - プログラムのエントリーポイント（開始点）は`func main()`です。この関数内に書かれたコードが実行されます。
  - **成果物:** `hello.go` を作成し、`go run`での実行と`go build`での実行ファイル生成を試す。

- **Day 3-4: 変数、データ型、制御構文**
  - **学習項目:**
    - **変数と定数の宣言:**
      - `var name string = "Taro"` のように型を明示して変数を宣言します。
      - `name := "Jiro"` のように `:=` を使うと型を推論して初期化できます（関数内でのみ使用可）。
      - `const Pi = 3.14` のように `const` で定数を宣言します。
    - **基本的なデータ型:**
      - `int` (整数), `float64` (浮動小数点数), `string` (文字列), `bool` (真偽値) などの基本的な型に触れます。
    - **制御構文:**
      - `if-else` (条件分岐), `for` (繰り返し), `switch` (多岐分岐) の基本的な使い方を学びます。Goの `for` は他の言語の `while` の役割も兼ねます。
  - **成果物:** 簡単な数当てゲームやFizzBuzz問題を解くプログラムを作成する。

- **Day 5-6: 配列、スライス、マップ**
  - **学習項目:**
    - **配列とスライス:**
      - **配列:** `[5]int` のように要素数を固定して宣言するデータ構造です。
      - **スライス:** `[]int` のように要素数が可変のデータ構造で、配列よりも遥かに頻繁に使われます。`append`で要素を追加し、`make`で初期容量を指定して作成できます。
    - **マップ:**
      - `map[string]int` のようにキーと値のペアを格納するデータ構造です。`make`で初期化し、`delete`で要素を削除します。
  - **成果物:** スライスやマップを使って、与えられた文章内の単語の出現回数をカウントするプログラムを作成する。

- **Day 7: Week 1 復習とミニプロジェクト**
  - **学習項目:** Week 1で学んだ知識の総復習。
  - **成果物:** 標準入力から受け取った複数の数値を合計・平均するCLIツールを作成する。

## Week 2: Goの特長的な機能（主戦力への橋渡し）

**目標:** Goらしいコードの書き方（関数、ポインタ、構造体、インターフェース）を理解する。

- **Day 8-9: 関数とエラーハンドリング**
  - **学習項目:**
    - **関数:**
      - `func add(x int, y int) int` のように引数と戻り値の型を定義します。
      - `func swap(x, y string) (string, string)` のように複数の値を返すことができます。これはGoの大きな特徴です。
    - **エラーハンドリング:**
      - Goには例外処理 (try-catch) がなく、関数の戻り値として`error`型を返すのが一般的です。
      - `result, err := someFunc()` のように値とエラーを同時に受け取り、`if err != nil` でエラーの有無を確認します。
  - **成果物:** 割り算を行い、ゼロ除算の場合にエラーを返す関数を作成し、呼び出し側で適切に処理する。

- **Day 10-11: ポインタと構造体**
  - **学習項目:**
    - **構造体 (`struct`):**
      - 複数のフィールド（データ）を一つにまとめたカスタムデータ型です。`type User struct { ID int; Name string }` のように定義します。
    - **ポインタ:**
      - メモリアドレスを保持する変数です。関数の引数に大きな構造体を渡す際のコピーコストを削減したり、関数内で引数の値を直接変更したりするのに使います。`&`でアドレスを取得し、`*`でそのアドレスが指す値（実体）にアクセスします。
  - **成果物:** `User` 構造体を定義し、ユーザー情報を更新する関数をポインタ渡しで作成する。

- **Day 12-14: メソッドとインターフェース**
  - **学習項目:**
    - **メソッド:**
      - 特定の型に関連付けられた関数です。`func (u User) GetName() string` のように、関数名の前に「レシーバー」と呼ばれる型を指定します。
    - **インターフェース:**
      - メソッドのシグネチャ（名前、引数、戻り値）の集まりを定義したものです。`type Shape interface { Area() float64 }` のように定義します。
      - ある型がインターフェースで定義されたメソッドをすべて実装していれば、その型はそのインターフェースを満たしていると見なされます（ダックタイピング）。
  - **成果物:** `Shape` インターフェースを定義し、`Rectangle` と `Circle` 構造体に実装する。

- **Day 15: Week 2 復習とミニプロジェクト**
  - **学習項目:** Week 2で学んだ知識の総復習。
  - **成果物:** 複数の図形の面積をまとめて計算するプログラムをインターフェースを使って作成する。

## Week 3: Webアプリケーションの基礎（稼げる主戦力）

**目標:** Goの標準ライブラリを使い、基本的なWebサーバーとAPIを構築できる。

- **Day 16-18: `net/http` パッケージ入門**
  - **学習項目:**
    - **Webサーバーの起動:** `http.ListenAndServe(":8080", nil)` を使って指定したポートでWebサーバーを起動します。
    - **リクエストハンドラ:** `http.HandleFunc("/", handlerFunc)` でURLのパスと処理関数を結びつけます。
    - **リクエストとレスポンス:** ハンドラ関数は引数として `http.ResponseWriter` (クライアントへの応答)と `http.Request` (クライアントからの要求) を受け取ります。
  - **成果物:** `/hello` にアクセスすると "Hello, Web!" と表示する簡単なWebサーバーを作成する。

- **Day 19-21: ルーティングとJSONの扱い**
  - **学習項目:**
    - **ルーティング:** `http.ServeMux` を使うと、URLのパスに応じて処理を振り分ける（ルーティングする）ロジックをより細かく制御できます。
    - **JSONの扱い:** `encoding/json` パッケージを利用します。構造体をJSON文字列に変換するのを「エンコード（マーシャリング）」、JSON文字列を構造体に変換するのを「デコード（アンマーシャリング）」と呼びます。
  - **成果物:** ユーザー情報のリストを返す `/users` というJSON APIエンドポイントを作成する。

- **Day 22: Week 3 復習とミニプロジェクト**
  - **学習項目:** Week 3で学んだ知識の総復習。
  - **成果物:** メモリ上に保持したTODOリスト（構造体のスライス）に対して、一覧取得 (`GET /todos`) と新規追加 (`POST /todos`) ができる簡単なAPIサーバーを作成する。

## Week 4 & 5: 実践的なアプリケーション開発（稼げる主戦力）

**目標:** DB接続、並行処理、テストを含む、プロダクトに近い形のアプリケーションを構築する。

- **Day 23-24: Go Modules と外部ライブラリ**
  - **学習項目:**
    - **Go Modules:** Goの公式な依存関係管理システムです。プロジェクトのディレクトリで `go mod init myapp` を実行して初期化します。
    - **外部ライブラリの利用:** `go get github.com/gorilla/mux` のようにコマンドを実行すると、外部で公開されているライブラリをダウンロードして利用できます。`go.mod`ファイルに依存関係が記録されます。
  - **成果物:** `gorilla/mux` などの高機能なルーターライブラリを使って、前週のTODOアプリのルーティングをリファクタリングする。

- **Day 25-26: データベース(SQL)入門**
  - **学習項目:**
    - **`database/sql` パッケージ:** GoでSQLデータベースを操作するための標準インターフェースです。
    - **ドライバ:** 実際に使用するデータベース製品（MySQL, PostgreSQL, SQLiteなど）に対応した「ドライバ」を別途インストールします。
    - **操作:** `sql.Open`で接続し、`db.Query`でデータを取得、`db.Exec`でデータを登録・更新・削除します。
  - **成果物:** TODOアプリのデータ保存先をメモリからSQLiteに変更し、永続化を実現する。

- **Day 27-28: 並行処理 (Goroutine と Channel)**
  - **学習項目:**
    - **Goroutine:** `go myFunction()` のように関数の前に `go` を付けるだけで、その関数を非同期に（現在の処理をブロックせずに）実行できます。非常に軽量なスレッドのようなものです。
    - **Channel:** ゴルーチン間で安全にデータを送受信するためのパイプです。`ch := make(chan int)` で作成し、`ch <- data` で送信、`data := <- ch` で受信します。
  - **成果物:** 複数の外部APIを並行して呼び出し、結果をチャネルで集約するサンプルプログラムを作成する。

- **Day 29: テストの基本**
  - **学習項目:**
    - **`testing` パッケージ:** Goの標準テストライブラリです。`*_test.go` というファイル名でテストコードを記述します。
    - **テスト関数:** `func TestMyFunction(t *testing.T)` という形式で関数を定義します。
    - **実行:** `go test` コマンドでプロジェクト内のテストを一括実行できます。
  - **成果物:** これまで作成したTODOアプリのハンドラやDBロジックに対してユニットテストを作成する。

- **Day 30: 総まとめ: CRUD APIの完成とドキュメント**
  - **学習項目:** 全知識を統合し、アプリケーションを完成させる。
  - **成果物:**
    - TODOアプリに更新 (`PUT /todos/{id}`) と削除 (`DELETE /todos/{id}`) 機能を追加し、CRUD APIを完成させる。
    - プロジェクトの概要、起動方法、API仕様を記述した `README.md` を作成する。

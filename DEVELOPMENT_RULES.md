# 開発ルール・プロセス

## Git運用ルール

### ⚠️ 重要: Git操作はユーザーが実施
- **Git関連の操作（commit, push, branch操作など）は必ずユーザーが実施する**
- AIは以下を実施しない：
  - `git add`
  - `git commit`
  - `git push`
  - `git branch`
  - `git merge`
  - その他のGitコマンド全般

### コミットのタイミング
- 各Dayのタスク単位でコミット
- ブランチは分けない（mainブランチで直接作業）

---

## ファイル配置ルール

### ドキュメント
- プロジェクトディレクトリの直下に格納
- 例: `PROJECT_OVERVIEW.md`, `DOMAIN_MODEL.md`

### 実装コード
- 後日、適切なディレクトリ構造を決定

---

## Day61-100 学習の進め方

### 各Dayの流れ
1. AIがその日のカリキュラム内容を提示
2. AIが成果物を作成（ドキュメント、コード等）
3. ユーザーが内容を確認・レビュー
4. **ユーザーがGitコミット** ← AIは実施しない
5. 次のDayへ進む承認

### 成果物の確認方法
- ファイルを直接確認
- 不明点や修正があればAIに依頼
